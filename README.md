1. Contexto e Objetivos

Este projeto investiga como aplicar Engenharia de Prompts ao desenvolvimento de dashboards web modernos, com foco em:

- geração de código (HTML, CSS e JavaScript)  
- modularização  
- boas práticas de UI/UX  
- heurísticas de usabilidade  
- acessibilidade  
- design systems  
- ferramentas modernas de front-end  

Foco do Caderno  
Aplicar engenharia de prompts para gerar, melhorar e documentar componentes de dashboards web, priorizando modularização, responsividade, UI/UX e consistência visual.

Objetivo  
Desenvolver a habilidade de criar prompts eficientes para gerar código limpo e modular, melhorar interfaces e acelerar o processo de desenvolvimento front-end com apoio de IA.

---

2. Curadoria de Fontes (NotebookLM)

As fontes selecionadas para embasar o estudo foram:

- Nielsen Norman Group – 10 Usability Heuristics  
- W3C – Accessibility Fundamentals

- Tailwind CSS + Vite

Essas referências fornecem uma base sólida entre UX, acessibilidade, design system, IA e desenvolvimento web moderno.

3. Engenharia de Prompts e Cicatrizes

### Prompt inicial
```
Explique como aplicar engenharia de prompts para gerar componentes de dashboards usando HTML, CSS e JavaScript, com base nas fontes carregadas.
```

### Refinamento
```
Reescreva a explicação com foco em modularização (HTML, CSS e JS separados) e boas práticas de UI/UX.
```

### Prompt avançado
```
Gere um componente de sidebar responsiva com dark mode e neon magenta, seguindo princípios de design de dashboards e boas práticas de acessibilidade. Separe o código em HTML, CSS e JS.
```

### Troubleshooting
```
Reescreva apenas o CSS garantindo responsividade, hierarquia visual clara, tokens de cor e sem redundâncias.
```

### Cicatrizes (aprendizados)
- A IA mistura HTML/CSS/JS se o prompt não for explícito.  
- Classes inexistentes surgem quando não há restrições claras.  
- Dark mode exige instruções diretas.  
- Prompts vagos geram código genérico.  
- Tokens de design aumentam a consistência.  
- UX só é respeitada quando mencionada explicitamente.

4. Miniguia de Estudo

### Resumo Estruturado
- Engenharia de prompts orienta modelos de IA a gerar resultados mais precisos.  
- No front-end, ela acelera a criação de componentes e melhora interfaces.  
- UX e acessibilidade são essenciais para dashboards profissionais.  
- Design systems garantem consistência visual.  
- Ferramentas como Tailwind, Vite e Semantic Kernel ampliam o potencial da IA.  

### Glossário

| Termo | Definição |
|-------|-----------|
| Prompt | Instrução dada à IA para gerar uma resposta. |
| Modularização | Separação de HTML, CSS e JS para organização e manutenção. |
| Heurísticas de Nielsen | Dez princípios de usabilidade para interfaces. |
| Acessibilidade (WAI) | Práticas para tornar interfaces utilizáveis por todos. |
| Design System | Conjunto de padrões visuais e componentes reutilizáveis. |
| Tokens de Design | Variáveis de cor, espaçamento, tipografia etc. |
| Prompt Refinement | Processo de melhorar um prompt até obter o resultado ideal. |
